# UI Prototype Specification

**Phase:** Prototyping
**Purpose:** Generate a detailed spec that AI vibe-coding tools can turn into a working prototype.

## Prompt Template

```
I need a web application prototype for [WHAT IT IS].

TARGET USERS: [who will use it and why]

CORE FUNCTIONALITY:
1. [Primary feature]
2. [Secondary feature]
3. [Tertiary feature]

KEY INTERACTIONS:
- [Interaction 1: what the user does and what happens]
- [Interaction 2: what the user does and what happens]
- [Interaction 3: what the user does and what happens]

DATA/CONTENT:
- [Describe realistic sample data to populate the prototype]
- [Specify ranges, types, and variety needed]

VISUAL STYLE:
- [Color scheme and mood]
- [Layout preferences]
- [Typography and readability notes]
- [Key visual indicators]

TECHNICAL NOTES:
- [Sorting, filtering, calculation requirements]
- [Responsive behavior]
- [Format specifications (currency, dates, etc.)]
```

## Tips

- The more specific your spec, the better the prototype
- Include realistic sample data — it makes the prototype feel real for user testing
- Specify visual indicators (colors for good/bad states, hover effects, etc.)
- See `examples/stock-portfolio-example.md` for a fully worked example
- Try this in Claude (artifacts), ChatGPT, Lovable, Replit, or Base44
