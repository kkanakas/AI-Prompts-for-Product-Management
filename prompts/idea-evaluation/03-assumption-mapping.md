# Assumption Mapping

**Phase:** Idea Evaluation
**Purpose:** Surface hidden assumptions your idea depends on, sorted by risk (high-impact, low-confidence first).

## Prompt Template

```
Feature idea: [your idea and the problem it solves]

What assumptions does this idea depend on?
Consider these categories:
1. User behavior (will they do X?)
2. User needs (do they actually want this?)
3. User retention (will they want to pay for it?)
4. Technical feasibility (can we build it as imagined?)
5. Market dynamics (outside conditions)
6. Resource availability (team, budget, time)

Sort these assumptions by high-impact, low-confidence first.
```

## Placeholders

| Variable | Description |
|---|---|
| `[your idea and the problem it solves]` | The feature concept and the user need it addresses |

## Tips

- Once you have the assumption list, design lightweight tests for the top 3-5
- Pair this with the Validation Questions prompt (04) to build an interview guide
